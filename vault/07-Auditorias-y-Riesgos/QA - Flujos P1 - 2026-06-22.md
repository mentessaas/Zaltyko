## Bugs encontrados y resueltos en esta sesion

### Bug A - `/api/reports/attendance/export` devolvia 500 por ZodError [CERRADO]

**Sintoma original**: `GET /api/reports/attendance/export?academyId=<uuid>` devolvia 500.

**Causa raiz**: doble problema en `src/app/api/reports/attendance/export/route.ts`:
1. `exportSchema` exigia `academyId: z.string().uuid()` por query string, pero `withTenant` ya lo resolvia desde la sesion como `context.tenantId`. Doble fuente de verdad.
2. `startDate`, `endDate`, `athleteId`, etc. se declaraban `z.string().optional()`, pero `URL.searchParams.get()` devuelve `null` (no `undefined`) cuando el param no existe. Zod no acepta `null` con `.optional()`.

**Fix aplicado** (2026-06-22):
- Quitar `academyId` del schema.
- Usar `context.tenantId` como fuente unica para `filters.academyId` y la query de nombre de academia.
- Cambiar todos los params opcionales a `z.string().nullable().optional()` (o `z.string().uuid().nullable().optional()`).

**Verificacion**: `curl /api/reports/attendance/export?format=pdf` con cookie de sesion devuelve HTTP 200 + PDF valido. Test E2E valida `Content-Type: application/pdf`.

### Bug B - `/app/[id]/athletes/[athleteId]/assessments` mostraba "Failed query" [CERRADO]

**Sintoma original**: `GET /app/<academyId>/athletes/<athleteId>/assessments` retornaba HTML con "Failed query". API `/api/assessments/<athleteId>` devolvia 500 con `INTERNAL_ERROR`.

**Causa raiz**: desincronizacion schema TS в†” DB. La tabla `athlete_assessments` en Supabase no tenia las columnas `assessment_type`, `total_score`, `tenant_id` que el schema TS en `src/db/schema/athlete-assessments.ts` declaraba (algunas como `notNull()`). Postgres devolvia:
```
error: column athlete_assessments.assessment_type does not exist
Perhaps you meant to reference the column "athlete_assessments.assessment_date".
```

**Fix aplicado** (2026-06-22):
- Creada migracion idempotente `supabase/migrations/20260622140000_sync_athlete_assessments_schema.sql` que:
  1. Crea el enum `assessment_type` si no existe (6 valores: technical, artistic, execution, coach_feedback, competition, practice).
  2. Agrega `assessment_type`, `total_score`, `tenant_id` a `athlete_assessments` con `ADD COLUMN IF NOT EXISTS`.
  3. Crea indices `athlete_assessments_type_idx` y `athlete_assessments_tenant_idx`.
- Migracion aplicada via `scripts/apply-migration.ts` (que envia el SQL completo de una sola vez, manejando bloques `$$ ... $$` correctamente).
- Verificado: `information_schema.columns` muestra las 3 columnas presentes con tipos correctos.

**Verificacion**: API devuelve `200 {data: [], meta: {total:0}}`. Pagina renderiza "Evaluaciones de [atleta]" sin "Failed query".
